# PRD - 旅遊網站系統

## 1. 專案概述
### 背景與動機
許多熱愛旅遊的人在規劃行程時，常因為需要跨多個平台查找景點、安排路線和計算預算而感到困擾。本專案旨在提供一個整合平台，讓使用者能一站式完成旅遊規劃，解決分散資訊帶來的痛點。

### 目標用戶
- 熱愛旅遊、喜歡自行規劃行程的自助旅行者。
- 負責安排家庭或團體旅遊的規劃者。

### 核心價值主張
讓旅遊規劃變得簡單直覺，整合景點探索、行程安排與預算控制，節省行前準備的時間，提升旅行體驗。

## 2. 功能需求
以下為系統的主要功能與對應的使用者故事：

### (1) 行程安排
- **使用者故事**：作為 **[旅遊規劃者]**，我希望 **[能夠建立多天的旅遊行程表並為每一天加入具體活動]**，以便 **[有條理地規劃並掌握我的旅行節奏]**。

### (2) 景點介紹
- **使用者故事**：作為 **[愛好探索的旅客]**，我希望 **[能夠搜尋並瀏覽各個景點的詳細介紹、照片與開放時間]**，以便 **[決定是否將該景點加入我的行程中]**。

### (3) 預算安排
- **使用者故事**：作為 **[重視花費的旅客]**，我希望 **[能夠為行程中的每個項目（如交通、住宿、餐飲、門票）設定預估花費，並自動計算總預算]**，以便 **[有效控制旅行開銷，避免超支]**。

### (4) 行程分享與共編
- **使用者故事**：作為 **[團體旅遊發起人]**，我希望 **[能夠透過連結將行程分享給旅伴，並允許他們共同編輯]**，以便 **[大家能一起討論行程並達成共識]**。

### (5) 旅遊心得與評價
- **使用者故事**：作為 **[熱心分享的旅客]**，我希望 **[能夠在行程結束後，對去過的景點留下評價與圖文心得]**，以便 **[提供其他使用者參考，建立社群互動]**。

## 3. 非功能需求
### 技術限制
- 後端框架：Flask
- 前端技術：HTML + Vanilla CSS + Jinja2 (需要支援 RWD 響應式設計)
- 資料庫：SQLite

### 效能與安全考量
- 密碼需經過加密（如 bcrypt）儲存。
- 防止常見網路攻擊，包含 SQL Injection 與 XSS（Jinja2 預設自動跳脫）。
- 網頁需確保在行動裝置與桌面版皆有良好的瀏覽體驗與載入速度。

## 4. MVP 範圍
### Must Have (必須具備)
- 使用者註冊與登入系統。
- 瀏覽景點介紹。
- 建立、檢視、編輯與刪除旅遊行程表。
- 預算安排與總計功能。

### Should Have (應該具備)
- 透過專屬連結分享行程（唯讀模式或共編模式）。

### Nice to Have (有了更好)
- 景點的評價與心得留言板。
- 結合地圖 API 顯示行程路線。
